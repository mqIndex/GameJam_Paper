# 2026-05-23 卡牌平衡改动总结

本次改动只动卡牌相关的数据/配置/商店/出牌循环，不涉及股价、情绪、突发事件主线规则。

---

## 一、新增的卡牌通用属性 (CSV 列)

cards.csv 新加 5 列，所有老卡补默认值：

| 列名                  | 类型 | 含义                                                                       |
| --------------------- | ---- | -------------------------------------------------------------------------- |
| `shop_unique`         | bool | 该卡是「全卡组限 1 张」。玩家牌组里只要存在 1 张，商店刷新池里就会跳过它。 |
| `shop_price`          | int  | 商店售价覆盖值。`>0` 时用此价，`=0` 时回落到默认 `SHOP_BUY_PRICE`。        |
| `daily_limit`         | int  | 每日使用次数上限。`>0` 时打到上限会被拦截，`=0` 表示不限。                 |
| `daily_exile`         | bool | 当日打出后封存：进弃牌堆但当日不会被洗回抽牌堆，跨日才解封。               |
| `discard_hand_redraw` | bool | 自动效果：打出后把当前手牌全部进弃牌堆，再抽相同数量。                     |

`Card` 运行时多一个临时状态 `daily_exiled`，用于记录「这张卡是不是已经在本日被封存」。

## 二、商店刷新规则改动

| 项               | 改前                       | 改后                                                |
| ---------------- | -------------------------- | --------------------------------------------------- |
| 单日卡牌数量     | 4                          | 6                                                   |
| 单次刷新内重复卡 | 已通过 `picked_ids` 去重   | 同上保持去重                                        |
| 玩家已拥有的过滤 | 无                         | `shop_unique=TRUE` 且玩家牌组里已有 → 池中直接剔除  |
| 商店售价         | 全部走 `SHOP_BUY_PRICE`    | 优先用 `card.shop_price`，未设置才回落默认价        |

刷新流程（玩家视角）：

1. 取所有 `in_shop=TRUE` 的 effect_id 作为基础池
2. 过滤：玩家牌组里已经有的 `shop_unique` 卡从池中拿掉
3. 用 `day` 作种子轮选，遇到重复 effect_id 跳过，最多取 6 张
4. 每张报价 = `card.shop_price > 0 ? shop_price : 默认价`
5. UI 端按报价显示并判断「能否买得起」

## 三、出牌循环新增的两类约束

### 1. 每日使用次数 (`daily_limit`)

- `GameState` 新增字典 `daily_play_count: effect_id → 当日已使用次数`
- 在「开始新一天」和「开新关卡」时清空
- `play_card` 在扣行动力之前检查：本卡 `daily_limit > 0` 且 `当前次数 ≥ daily_limit` 时直接拦下，给玩家日志提示
- 成功打出之后计数 +1

### 2. 当日封存 (`daily_exile`)

- 卡走正常出牌流程进入弃牌堆，但同时被打上 `daily_exiled = true`，并日志「[封存] xxx 当日不再洗回牌堆」
- 当抽牌堆抽空需要洗牌时：
  - 弃牌堆按 `daily_exiled` 拆两堆
  - 未封存的洗回抽牌堆
  - 封存的留在弃牌堆，日志同时显示「封存 N 张留在弃牌堆」
- 跨日（离开商店进入下一天）时：把所有牌的 `daily_exiled` 一律清掉，再合并洗牌
- 因为封存走的是「卡上的运行时状态位」而不是 effect_id 黑名单，「顺势而为」「计划得当」「化整为零」等其他卡的弃 / 抽 / 移动操作都不会让封存卡脱离弃牌堆，达到「不被其他卡效果影响」的语义

## 四、单卡数值/规则调整

| 卡名     | effect_id        | 改动                                                                                                                              |
| -------- | ---------------- | --------------------------------------------------------------------------------------------------------------------------------- |
| 内幕消息 | `insider_basic`  | `in_shop=TRUE`、`shop_unique=TRUE`、`shop_price=5000`、`daily_limit=1`、`daily_exile=TRUE`；描述更新；旧的 `insider_plus` 退出商店 |
| 小买一笔 | `small_buy`      | `shop_unique=TRUE`（全卡组限 1 张，购入后商店不再上架）                                                                           |
| 小卖一笔 | `small_sell`     | 同上                                                                                                                              |
| 舆论反转 | `sentiment_flip` | `shop_unique=TRUE`（全卡组限 1 张，购入后商店不再上架）                                                                           |
| 风云变幻 | `reroll_event`   | `daily_limit=1`（每日只能用 1 次）                                                                                                |
| 化整为零 | `shatter_card`   | `shop_price=5000`（商店售价覆盖默认价）                                                                                           |
| 快速换手 | `quick_swap`     | 新卡：SKILL，2 费，弃光手牌中其他牌再抽相同数量；`in_starter=TRUE` 起手牌组带 1 张                                                |
| 孤注一掷 | `mob_card`       | 新卡：SKILL，2 费，`in_shop=TRUE`、`shop_unique=TRUE`；股价 ±X%（50/50），X = 当前牌组（手牌+抽牌堆+弃牌堆）中 BUY+SELL 总数      |

### 孤注一掷 与 化整为零 的交互

X 直接取自 `get_full_deck()`，对 `c.is_buy() or c.is_sell()` 计数，不过滤 `transient`。所以化整为零产生的 `small_buy`/`small_sell`（本回合限定的 transient 小卡）只要还在手牌/抽牌堆/弃牌堆里，每张都会被孤注一掷计入 X。同回合先 shatter 再打孤注一掷时，X 会被显著放大。
