## 一、规则层改动（`Project/scripts/game_state.gd` + `card_database.gd`）

### 1. 行动力规则

- 行动力上限从 3 改为 6
- 行动力第一回合为3，然后第二回合是4，第三回合是5，第四回合是6，第五回合是6，第六回合是6


### 2. 首回合"抽到 7 张"bug 修复

- 起因：原实现"先抽 6 → 缺类型再 `hand.append()` 补一张"，缺技能时会变 7 张
- 改法：整局第一回合改为**先种 1 买 + 1 卖 + 1 技能，再补到 6 张**
- 保证：第一回合手牌恒为 6，且三类齐全

### 3. 初始卡组配比

- 旧：3 买 + 3 卖 + 3 内幕 + 3 大V吹票（4×3）
- 新：**5 买 + 5 卖 + 1 大V吹票 + 1 内幕消息**，总数仍 12 张

### 4. 买卖牌效果重写

| 卡 | 旧效果 | 新效果 |
|----|--------|--------|
| 买入 | 消耗现金 10% 买入，不影响股价 | **固定买 100 股，股价 +1%** |
| 卖出 | 卖出持仓 10%，不影响股价 | **固定卖 100 股，股价 -1%** |

- 新增 `_buy_shares(n)`（**顺便修了原代码调用一个不存在函数的 nil bug**）
- `play_card` 加资源前置检查：
  - 买入要求 `cash ≥ 100 × price`
  - 卖出要求 `shares ≥ 100`
  - 不满足直接拒绝出牌，**不扣 AP、不进弃牌堆**
- 卡牌描述文案同步更新



### 6. 每天重置市场情绪

- `_start_day` 开盘时把 `bull / bear` 拉回初始中性值（`INITIAL_BULL = 50` / `bear = 50`）
- 上一天累计的利好/利空情绪不会带到新一天

### 7. 默认股价震荡（恢复，按策划 4.3）

- `_settle_turn` 回合末重新滚一次自然波动
- `drift ~ Normal(μ, σ)`，`μ = (bull - 50) / 50 × 10%`，`σ = 1.2%`（暂定，策划 4.3 标注待数值组确认）
- `NATURAL_DRIFT_CLAMP` 由 3% 改为 **10%**，对齐策划
- 调用 `apply_price_change(drift, true)`：传 `ignore_emotion_modifier=true`，因为 μ 本身已经体现情绪方向，不再叠加情绪倍率
- 生成一根 `kind:"settle"` 分时 K，写日志 `回合末自然波动 +x.xx% → ¥xxx.xx`

### 8. BGM 接入

- `_ready` 末尾启动 `AudioStreamPlayer` 播放 `res://assets/bgm/Measured Inflection.mp3`
- 把 `AudioStreamMP3.loop` 置 true 实现循环
- 初始音量 -6dB，节点挂在主场景上随场景生命周期销毁
- 需 Godot 编辑器首次打开生成 `.import`，否则 `load()` 失败

## 五、Phase 2 / Phase 3 累积改动





### 3. 突发事件系统重写（25 条）

事件数据/逻辑从 `game_state.gd` 拆出，模仿 `card.gd` / `card_database.gd` 风格新增：

- `Project/scripts/event.gd` — `Event` 数据类（RefCounted），含 `Category` 枚举 (GOOD/BAD/NEUTRAL) + 字段 + `is_*()` / `category_str()` 辅助
- `Project/scripts/event_database.gd` — `EventDatabase` 静态工厂，提供 `build_event_pool()` / `make_by_id()` / `all_event_ids()`

事件池扩到 25 条（原 8 条被替换）：

| 类型 | 事件 |
|------|------|
| 利好情绪 | 降息 +10 / 外盘暴涨 +20 / 黄金时代 +30 |
| 利空情绪 | 加息 -10 / 外盘暴跌 -20 / 战争风险 -30 |
| 股价直击 | 盈利双击 +20% / 黑天鹅 -20% |
| 情绪锚定 | 信仰充值（≥50）/ 信仰崩塌（≤50） |
| 持续 3 回合 | 超预期财报（+5 且涨幅 ×1.2）/ 财报逆袭（-5 且跌幅 ×1.2） |
| 长期修饰 | 火线预期 / 下行预期 / 杠杆涨市（成交价格 ×1.5）/ 风险警示（情绪倍率 ×0.5）/ 市场失真（脱钩） |
| 涨跌停 | 限制涨停 ≤5% / 限制跌停 ≤5% |
| 牌库 / AP | 账户审查（每回合冻结 1 张）/ 重点监控（每回合最多 2 张技能）/ 混乱之日（每回合 50% AP+1 / 50% AP-1） |
| 其他 | 神秘资金（回合末 30% 概率 ±5%）/ 意外事件（情绪随机 ±10） |

### 4. 事件 modifier 通用机制

`game_state.gd` 新增状态：`event_modifiers: Dictionary` / `event_modifier_dur: int` / `triggered_event_ids_this_level: Dictionary` / `skills_played_this_turn: int`。

各 modifier 落地点：

| modifier 键 | 落地位置 | 行为 |
|-------------|----------|------|
| `card_price_up_mul` / `card_price_down_mul` | `_dispatch_effect` 技能牌方向倍率 | 价格变化方向放大 |
| `card_trade_price_mul` | `_dispatch_effect` 买/卖牌 | 成交引发的价格变化 ×1.5 |
| `emotion_modifier_mul` | `_emotion_modifier_for_price` 末尾 | 情绪倍率整体 ×0.5 |
| `decouple` | `_emotion_modifier_for_price` 返回 1.0 + `_roll_natural_drift` 中 μ 置 0 | 卡牌与自然波动均与情绪脱钩 |
| `emotion_floor` / `emotion_ceiling` | `apply_emotion_delta_bull` clamp | 当时段情绪上下限 |
| `cap_drift_up` / `cap_drift_down` | `apply_price_change` 末尾 | 累计涨/跌幅相对回合开盘价封顶 |
| `freeze_per_turn` | `_start_turn` | 每回合随机弃 N 张手牌进弃牌堆 |
| `skill_cap_per_turn` | `play_card` 前置检查 | 本回合技能牌使用上限 |
| `mystery_active` | `_settle_turn` 自然波动叠加 | 30% 概率额外 ±5% |
| `ap_chaos` | `_start_turn` 末尾 | 每回合 50% AP+1 / 50% AP-1（含触发回合） |

### 5. 事件刷新规则

- 每天第 1 / 第 5 回合开盘抽牌后调用 `_trigger_random_event()`
- 一周内同事件不重复：抽样前按 `triggered_event_ids_this_level` 过滤；整池跑完后兜底退回完整池
- 新事件刷新时清空上一事件的 `banned_effect_ids` / `event_modifiers` / `event_modifier_dur`
- 短期事件用 `dur_turns`（如超预期财报 / 财报逆袭 = 3）；`_settle_turn` 末尾倒计时，归零时清空并发 `event_triggered.emit(null)` 让 UI 复位
- `_start_day` 清当天事件残留；`new_level` 同时清 `triggered_event_ids_this_level`（"一周不重复"等于一关不重复）

### 6. 事件 UI

- 事件触发弹窗：`AcceptDialog`，标题"突发事件 · 利好/利空/中性"按 `category` 染色，正文 = 事件名 + 描述 + 效果文案；点"知道了"关闭
- 顶栏"今日事件"按钮颜色：利好红 / 利空绿 / 中性金 / 空灰；dur_turns 到期发空载荷时自动复位
- 鼠标悬停按钮弹出下拉信息栏：
  - `event_tip_panel: PanelContainer`（顶层 `top_level=true`、`z_index=100`、`mouse_filter=IGNORE`）
  - 三行：标题（带"[利好/利空/中性]"标签）+ 描述（autowrap）+ 效果（autowrap）
  - 无事件时显示"暂无突发事件，每天第 1 / 第 5 回合刷新"
